# Diansai25_G
25年电赛G题
2025年全国大学生电子设计竞赛G题 电路模型探究装置

项目概述
本项目基于STM32F4微控制器，实现了正弦信号生成、已知模型电路的定幅输出控制，以及未知模型电路的自主学习与滤波类型识别，完成了电赛G题的基本要求与发挥部分一。项目通过AD9833波形发生器生成可调频率和幅度的正弦信号，利用ADC采集电路输出数据，结合串口屏实现人机交互，可快速完成系统辨识与滤波类型判断。

硬件平台
主控核心：STM32F4系列微控制器
波形生成模块：AD9833可编程波形发生器，用于生成高精度正弦信号
信号采集模块：内置ADC，配合信号调理电路采集输出电压峰峰值
人机交互模块：串口显示屏（HMI），用于功能选择、参数显示与设置
按键模块：独立按键，用于辅助功能切换
已知模型电路：根据题目传递函数搭建的二阶电路，用于验证定幅输出功能
电源模块：为各模块提供稳定电源，数字与模拟电路分开供电以降低噪声

软件环境
开发工具：Keil MDK 5
编程语言：C语言
固件库：STM32 HAL库
辅助工具：STM32CubeMX（用于初始化配置）

代码结构
main.c：包含主程序逻辑、各功能模块初始化与调用，实现波形生成、幅度控制、系统辨识等核心功能
ad9833.c/h：AD9833驱动代码，实现GPIO初始化、频率设置、幅度设置等功能
AD9833_2.c/h：备用AD9833驱动代码，提供不同的引脚配置与实现方式
interrupt.c/h：中断处理代码，包含定时器中断（按键扫描、ADC采样触发）、ADC转换完成中断、串口接收中断
HMI.c/h：串口屏通信代码，封装了发送字符串、数字、浮点数的函数，用于与串口屏交互
myusart.c/h：串口打印封装代码，实现类似printf的功能，方便调试与数据传输
delay.c/h：延时相关函数，用于时间间隔判断
TypeDef.h：常用数据类型定义，方便代码移植与阅读

功能实现
基本要求
1. 正弦信号生成：通过AD9833生成频率可调的正弦信号，频率范围覆盖100Hz至1MHz，步长支持100Hz调整，输出电压峰峰值最大可达3V以上。
2. 已知模型电路定幅输出：预先通过实验测量不同频率下，使已知模型电路输出1~2V（步长0.1V）所需的输入幅度值，存储在Data数组中。运行时根据设定的频率和目标输出幅度，查表设置AD9833的输出幅度，实现定幅控制。
3. 人机交互：通过串口屏选择不同功能模式，显示当前频率、输出幅度等参数，支持通过屏幕指令或按键调整参数。

发挥部分一
未知模型电路系统辨识：采用扫频法，从低到高输出不同频率的正弦信号，同时通过ADC采集未知模型电路的输出电压峰峰值，存储幅频特性数据。使用classify_by_passband函数对数据进行分析，根据幅频特性的变化趋势判断滤波类型（低通、高通、带通、带阻），并在串口屏上显示结果。

使用说明
硬件连接
1. 按照电路原理图连接AD9833模块、串口屏、按键、已知模型电路等，确保电源与信号链路连接正确。
2. 检查模拟地与数字地的连接，确保仅单点共地，降低噪声干扰。
3. 为系统提供稳定的5V电源供电。

软件编译与下载
1. 使用Keil MDK 5打开工程文件，确认编译环境配置正确。
2. 编译工程，生成.hex可执行文件。
3. 通过ST-Link或J-Link仿真器，将.hex文件下载到STM32主控芯片中。

操作方式
1. 上电后，系统自动初始化各模块，串口屏显示主界面。
2. 通过串口屏选择功能模式：
   基本要求模式：可设置正弦信号频率，观察输出波形；或设置已知模型电路的目标输出幅度，验证定幅控制功能。
   发挥部分一模式：连接未知模型电路，启动系统辨识，等待扫频完成后查看滤波类型识别结果。
3. 可通过串口屏指令或按键调整频率、幅度等参数，具体操作参考串口屏界面提示。

注意事项
1. AD9833的频率计算基于25MHz晶振，若使用不同频率的晶振，需修改ad9833.c中的频率计算参数。
2. Data数组中的幅度值是基于特定硬件电路测量得到的，若更换硬件或调整电路参数，需重新测量并更新数组数据。
3. 系统辨识时，需确保未知模型电路连接正确，扫频过程中避免外界干扰，保证采集数据的准确性。
4. 串口屏的通信协议需与HMI.c/h中的函数匹配，若更换串口屏型号，需调整通信指令格式。
5. 代码中部分参数（如ADC采样率、扫频范围等）可根据实际需求调整，调整时需注意各模块的时序配合。
